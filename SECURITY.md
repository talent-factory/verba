# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest release | Yes |
| Older releases | No |

Only the most recent published version of Verba receives security updates. Users are encouraged to keep their installation up to date.

## Reporting a Vulnerability

If you discover a security vulnerability in Verba, please report it responsibly. **Do not open a public GitHub issue for security-related concerns.**

Instead, send an email to **info@talent-factory.xyz** with the following information:

1. A description of the vulnerability and its potential impact.
2. Steps to reproduce the issue, if applicable.
3. The version of Verba and VS Code in which the vulnerability was observed.
4. Any suggested remediation, if you have one.

We will acknowledge receipt within 48 hours and provide a substantive response within five business days. We ask that you refrain from disclosing the vulnerability publicly until a fix has been released.

## Scope

This policy covers the Verba VS Code extension and its published source code. It does not cover third-party services (OpenAI, Anthropic) whose APIs the extension consumes. Vulnerabilities in those services should be reported to their respective maintainers.

## Acknowledgements

We appreciate the efforts of security researchers and contributors who help keep Verba and its users safe. With your permission, we will credit you in the release notes of the fix.
